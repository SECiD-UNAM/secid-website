## Codebase Understanding

### Architecture Overview

Astro 4.x static output with React islands (`client:load` or `client:only="react"`). Firebase Hosting serves static files from `dist/client/`. Dynamic routes use Firebase rewrites to static fallback pages (e.g., `/es/companies/*` → `/es/companies/profile/index.html`). React components read params from URL client-side.

### Key Patterns Discovered

- **Astro SSR vs Static**: `output: 'static'` means `prerender = false` pages don't work on Firebase Hosting. Use static fallback pages + Firebase rewrites instead.
- **Firestore rules hasOnly**: Company creation was silently failing because the rules `hasOnly` allowlist didn't include `slug` and `memberCount`. Any new field added to a collection MUST be added to the rules allowlist.
- **Auth in React islands**: `client:only="react"` components don't render server-side. Auth state loads async via `onAuthStateChanged`. Always handle loading/not-ready states.
- **Company creation**: Direct Firestore writes require `isVerified()`. The `/api/companies` route is more reliable since it handles auth server-side.
