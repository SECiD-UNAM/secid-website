Company Network Directory — 3 Approaches

A page for members to discover where other members work (and have worked), find references, and connect

A

Company Cards with Member Drawer

Grid of company cards (logo + name + member count). Click a company to open a slide-out drawer showing current and former members with contact buttons. Filter by industry, search by name. Simple, focused.

Pros

  • Clean, familiar UX
  • Fast to build — reuses CompanyLogo + existing data
  • Mobile-friendly

Cons

  • Less visual impact
  • No network visualization
B

Interactive Network Map

Visual graph showing companies as nodes connected to member nodes. Click a company to see who works/worked there. Hover for quick info. Timeline slider to see how the network evolved over time. Think LinkedIn's "company insights" but visual.

Pros

  • Wow factor — visually impressive
  • Shows connections between companies (shared members)
  • Timeline reveals career migration patterns

Cons

  • Complex to build (graph library needed)
  • Can be confusing with many nodes
  • Poor on mobile
C

Company Profiles + Reference System

Each company gets a dedicated profile page (/companies/{slug}) showing: logo, description, industry, current members, alumni (former members), and a "Request Reference" button. Main page is a searchable/filterable company list. Members can mark themselves as "available for references" at each company.

Pros

  • Most complete — dedicated company pages
  • Reference request system adds real value
  • SEO-friendly (public company pages)
  • Scales well with data

Cons

  • Most work to build
  • Needs moderation for company descriptions

Which approach resonates? You can also combine elements — e.g., C's company profiles with B's network visualization as an optional view.
