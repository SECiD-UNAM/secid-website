### Changed

- **Universal Listing System — Task 15: ResourceLibrary migration**: Migrated `src/components/resources/ResourceLibrary.tsx` to use `useUniversalListing` hook with `ClientSideAdapter`. The adapter's `fetchAll` calls `searchResources` with optional `initialCategory` constraint, loads up to 100 resources, and returns them for client-side processing. Category, type, and difficulty filters defined as `FilterDefinition[]` and wired to `ListingFilters` in collapsible mode. Custom domain features preserved: bookmarks tab (client-side filter by `bookmarkedIds` Set), popular/recent tabs (change `setSort` via `handleTabChange`), category grid (hidden once a filter is active), stats bar, and upload flow. `lang` prop added. Default config: `defaultViewMode: 'grid'`, `paginationMode: 'offset'`, `filterMode: 'collapsible'`. Adds full-text search across title, description, summary, tags, searchKeywords, and author name. 19 unit tests, all passing.

- **Universal Listing System — Task 14: EventList migration**: Migrated `src/components/events/EventList.tsx` to use `useUniversalListing` hook with `ClientSideAdapter`. The adapter's `fetchAll` runs the Firestore `getDocs` query (published events, ordered by startDate, limit 50) and returns the full dataset; the adapter handles client-side full-text search (across title, description, tags via `toSearchable`) and type filter (equality match on `type` field). Time filter (upcoming/past/all) applied post-hook via `useMemo` since it requires date comparison against `new Date()`. Calendar view kept as custom domain code outside the universal system, toggled by a new "Listing / Calendar" tab bar. Grid and list view modes connected to `ListingViewToggle`. Offset pagination via `ListingPagination`. Loading state uses `ListingLoading`, empty state uses `ListingEmpty`, search uses `ListingSearch`, type dropdown uses `ListingFilters`. Props interface (`lang`) unchanged; `defaultViewMode: 'grid'`. 13 unit tests added.

- **Universal Listing System — Task 13: BlogList migration**: Migrated `src/components/blog/BlogList.tsx` to use `useUniversalListing` hook with `ClientSideAdapter`. The adapter's `fetchAll` runs `getBlogPosts` + `mergeBlogPosts` + `filterByLocale` to produce the full locale-correct dataset; the adapter handles client-side search (across title, excerpt, content, authorName, category), category filtering, and cursor-based pagination. Domain-specific features kept as custom code: category tab UI, featured post hero (shown only when `activeCategory === 'all'` and no search query), blog card markup, and newsletter CTA. Loading state uses `ListingLoading`, empty state uses `ListingEmpty`, search input uses `ListingSearch`, load-more uses `ListingPagination`. Props interface (`lang`, `initialPosts`) unchanged. 13 unit tests added.

- **Universal Listing System -- Task 12: MemberDirectory migration**: Migrated `src/components/directory/MemberDirectory.tsx` to use `useUniversalListing` hook and building-block components (`ListingViewToggle`, `ListingGrid`, `ListingList`, `ListingCompact`, `ListingPagination`, `ListingEmpty`, `ListingLoading`, `ListingStats`). Props interface unchanged.

- **CV PDF generator**: Replaced `src/lib/cv/pdf-generator.ts` with a polished version ported from the personal site generator. New features: running headers on page 2+ (name, title, email), accent-color bullet points parsed from hex, text sanitization for unicode/spaced-out text, proper date formatting (YYYY-MM to "Jan 2024"), format-specific content slicing (full/resume/summary), centered page numbers ("Page X / Y"), section headers with uppercase text and partial underline, two-column skills layout, company name in accent/italic, technology tags on experience items. ### Security

- **TD-002/TD-003**: Fixed 5 Astro CVEs by upgrading to v5.18.0: GHSA-ggxq-hp9w-j794 (middleware auth bypass via URL encoding), GHSA-whqg-ppgf-wp8c (double URL encoding auth bypass), GHSA-wrwg-2hg8-v723 (reflected XSS via server islands), GHSA-qq67-mvv5-fw3g (SSRF via Host header injection), GHSA-x3h8-62x9-952g (dev server file read).

Added `TWO_FACTOR_AVAILABLE` feature flag (set to `false`), all 2FA operations now throw "not yet available" errors, and the Security Settings UI shows a "Coming Soon" banner instead of fake enable/disable controls. Two-layer defense: (1) middleware now includes `/api/create-` in protected paths and rejects requests without a valid session, (2) each endpoint performs its own `verifyRequest()` check as defense-in-depth.
